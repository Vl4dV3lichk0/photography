# Telegram-бот для записи на фотосессии (aiogram 3)

Функции MVP:
- Несколько админов по Telegram ID
- Города задает только админ
- Расписание по городам и датам (почасовые слоты)
- Блокировки времени
- Клиентская запись: город -> месяц -> день -> часы
- Несколько последовательных слотов в одной заявке
- Подтверждение заявки админом
- Отмена с уведомлением обеих сторон
- Напоминания за 24 часа и 2 часа
- Ежедневная сводка админам
- Согласие на обработку персональных данных
- Аудит событий по записям
- Глобальный прайс за 1 час и расчет финальной суммы при бронировании
- Автоархивация прошедших съемок со статусом `archived`
- Архивные JSON-снимки с ручным скачиванием и удалением

## Запуск

1. Установите зависимости:
```bash
cd bot
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2. Скопируйте переменные окружения:
```bash
cp .env.example .env
```

3. Настройте `.env`:
- `BOT_TOKEN`
- `OWNER_TG_ID`
- `ADMIN_IDS`
- `DATABASE_URL`
- `TIMEZONE`
- `POLICY_VERSION`

4. Запустите бота:
```bash
python -m app.main
```

## Диагностика (если бот не отвечает)

Запустите health-check:
```bash
python scripts/health_check.py
```

Скрипт проверяет:
- загрузку `BOT_TOKEN` и `DATABASE_URL`
- доступ к Telegram API (`getMe`, `getWebhookInfo`)
- подключение к PostgreSQL
- наличие всех таблиц схемы
- количество админов в таблице `admins`

## Команды

- `/start` - клиентское меню и превью расписания
- `/my` - мои записи
- `/admin` - панель администратора
- `/price` - прайс
- `/archive_now` - ручной запуск архивации (админ)

## Форматы ввода в админ-панели

- Дата: `YYYY-MM-DD`
- Рабочее окно/блокировка: `10-18`

## Важно

- На одну дату можно публиковать расписание только для одного города.
- Для записи можно выбрать только последовательные часы.
